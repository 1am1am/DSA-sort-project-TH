\subsection[Quick Sort]{Quick Sort}
    \subsubsection{Overview}
        \begin{itemize}
            \item \textbf{History and Origin:} Developed by Tony Hoare in 1959. One of the most famous divide-and-conquer algorithms.
            \item \textbf{Efficiency:} Highly efficient and generally outperforms other $O(n \log n)$ algorithms in practice due to excellent cache locality.
            \item \textbf{Application:} The default sorting algorithm in many standard libraries (e.g., C++ \texttt{std::sort}).
        \end{itemize}
    \subsubsection{Idea}
        \begin{itemize}
            \item It works by selecting a 'pivot' element from the array and partitioning the other elements into two sub-arrays according to whether they are less than or greater than the pivot.
            \item The sub-arrays are then sorted recursively.
            \item The base cases are empty arrays or arrays of a small size, where a simpler algorithm (like Insertion Sort) is often used to finish sorting.
        \end{itemize}

    \subsubsection{Step-by-Step Algorithm}
        \begin{itemize}
            \item \textbf{Step 1:} If the array segment has less than 10 elements, apply \textbf{Insertion Sort} to quickly sort it and terminate the recursion.
            \item \textbf{Step 2:} Select a pivot using the \textbf{Median-of-Three} method (finding the median of the first, middle, and last elements).
            \item \textbf{Step 3:} Swap the chosen pivot with the second-to-last element to temporarily move it out of the way.
            \item \textbf{Step 4:} Use two pointers, \textit{left} and \textit{right}, starting from the ends of the segment.
            \begin{itemize}
                \item Move \textit{left} forward until finding an element larger than the pivot.
                \item Move \textit{right} backward until finding an element smaller than the pivot.
                \item Swap these two elements if \textit{left} $\leq$ \textit{right}.
            \end{itemize}
            \item \textbf{Step 5:} Restore the pivot to its correct final position.
            \item \textbf{Step 6:} Recursively apply Quick Sort to the sub-array before the pivot and the sub-array after the pivot.
        \end{itemize}
        
    \subsubsection{Pseudo Code}
        \begin{itemize}
            \item \textbf{Algorithm:} Quick Sort (Optimized)
            \item \textbf{Input:} An array A, left index l, right index r
        \end{itemize}
        \begin{verbatim}
        1. If (r - l) < 10:
        2.    InsertionSort(A, l, r)
        3.    Return
        4. p = Partition(A, l, r) // Uses Median-of-Three
        5. QuickSort(A, l, p - 1)
        6. QuickSort(A, p + 1, r)
        \end{verbatim}
    \subsubsection{Complexity}
        \begin{itemize}
            \item \textbf{Time Complexity:}
            \begin{itemize}
                \item Best case: $O(n \log n)$
                \item Average case: $O(n \log n)$
                \item Worst case: $O(n^2)$ 
                    \subitem Occurs when the partition is highly unbalanced, though the Median-of-Three optimization makes this extremely rare.
            \end{itemize}
            \item \textbf{Space Complexity:} $O(\log n)$ 
                \subitem Bounded by the call stack depth during recursion.
        \end{itemize}
        
   \subsubsection{Variants and Optimizations}
        \begin{itemize}
            \item \textbf{Implemented Optimization 1: Median-of-Three Pivot Selection}
            \begin{itemize}
                \item \textbf{Implementation:} The \texttt{selectPivot} function in the provided C++ code chooses the median of the first, middle, and last elements ($A[l], A[mid], A[r]$) to use as the pivot.
                \item \textbf{Benefit:} Avoids the $O(n^2)$ worst-case performance on already sorted or reverse-sorted arrays, ensuring a more balanced partition in practical scenarios.
            \end{itemize}
            \item \textbf{Implemented Optimization 2: Hybrid Sorting (Insertion Sort Threshold)}
            \begin{itemize}
                \item \textbf{Implementation:} The \texttt{quickSortArray} function checks if the sub-array size is less than 10 (\texttt{(r - l) < 10}). If so, it halts recursion and delegates the sorting to \texttt{insertionSortSubmatrix}.
                \item \textbf{Benefit:} Eliminates the overhead of recursive function calls for very small arrays, where simple $O(n^2)$ algorithms like Insertion Sort execute much faster due to lower constant factors.
            \end{itemize}
        \end{itemize}
