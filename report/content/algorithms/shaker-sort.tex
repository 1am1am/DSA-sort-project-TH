\subsection[Shaker Sort]{Shaker Sort}
    \subsubsection{Overview}
        \begin{itemize}
            \item \textbf{History and Origin:} A variant algorithm from bubble sort - which named as cocktail sort, coctail-shaker sort, bidirectional bubble sort,etc.
            \item \textbf{Efficiency:} Best for small datasets or nearly sorted/sorted data. Worst for reverse-sorted datasets.
            \item \textbf{Application:} Using shaker sort for better efficiency than bubble sort. It is also uses for teaching basic algorithm about sorting or use for sort a small set data.
        \end{itemize}
    \subsubsection{Core concepts}
        \begin{itemize}
            \item The shaker sort compares adjacent elements and swaps their position if neccessary.
            \item The algorithm works to swap the larger items to the end of the array and swap the smaller items to the begin of the array, those items are described as bubbles float to the surface and shrink to the bottom.
        \end{itemize}
    \subsubsection{Step-by-step Expanation}
        \begin{itemize}
            \item Assume that the array of n elements, a[0...(n-1)].
            \item \textbf{Step 1: } Set \textit{start} = 0, \textit{end} = n - 1 and \textit{pos} = n - 1
            \item \textbf{Step 2: } If start = end then stop algorithm else do all the following steps below.
            \item \textbf{Step 3: } Set pos = end.
            \item \textbf{Step 4: } Run a loop from start to end, if there is an adjacent pair which not ascending then swap them and set pos = less index of that pair.
            \item \textbf{Step 5: } Set end = pos.
            \item \textbf{Step 6: } Run a loop from end to start, if there is an adjacent pair which not descending then swap them and set pos = larger index of that pair.
            \item \textbf{Step 7: } Set start = pos, then repeat \textbf{Step 2}
        \end{itemize}

    \subsubsection{Pseudo Code}
        \begin{itemize}
            \item \textbf{Algorithm:} Shaker Sort
            \item \textbf{Input:} An array A of n elements
        \end{itemize}
        \begin{verbatim}
        1. l ← 0
        2. r ← length(A) - 1
        3. while l < r 
        4.      pos ← r
        5.      for i ← l+1 to r
        6.           if A[i] < A[i-1]
        7.                swap A[i], A[i-1]
        8.                pos ← i-1
        9.      r ← pos
        10.     for i ← r-1 downto l
        11.          if A[i] > A[i+1]
        12.               swap A[i], A[i+1]
        13.               pos ← i+1
        14.     l ← pos
        \end{verbatim}
    \subsubsection{Complexity}
        \begin{itemize}
            \item \textbf{Time Complexity:}
            \begin{itemize}
                \item Best case: $O(n)$
                    \subitem When the input is already sorted.
                \item Average case: $O(n^2)$
                    \subitem When the input is random.
                \item Worst case: $O(n^2)$ 
                    \subitem When the input is reverse-sorted.
            \end{itemize}
            \item \textbf{Space Complexity:} O(1) 
                \subitem In-place algorithm. Requires only a constant amount of extra memory.
        \end{itemize}
        
    \subsubsection{Variants and Optimizations}
        \begin{itemize}
             \item \textbf{Origin implementation: } Origin shaker sort
            \begin{itemize}
                \item Compares all adjacent pair of elements for twice times from go forward and backward.
                \item Optional optimize: After each loops, minimize start and end by one cause we know all the values from begin and end were in their correct position.
                \item Additional flag, if they do not swap any times, break the loop.
            \end{itemize}
            \item \textbf{Improved implementation: } Shaker sort with last swapped
            \begin{itemize}
                \item Having all the structures from original shaker sort but has below optimize.
                \item Additional variable \textit{pos} stores the change of position which was the place swapped in array. 
                \item We only care about that pos cause if we go forward, all the index before that pos were in their correct position, it is also correct for backward. So we can minimize the range of array.
            \end{itemize}
        \end{itemize}
