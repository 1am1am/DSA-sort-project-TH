\subsection[Merge Sort]{Merge Sort \cite{balogun2023halstead}}
    \subsubsection{Overview}
        \begin{itemize}
            \item \textbf{History and Origin:} Merge Sort is a divide-and-conquer algorithm that was invented by John von Neumann in 1945.
            Merge sorting is also known as a divide and conquer method of sorting features, and it is based on this method (Varshney \& Yadav, 2014).
            \item \textbf{Efficiency:} Consistently performs well on large datasets.
            \item \textbf{Application:} Commonly used in external sorting and when stable sorting is required.
        \end{itemize}
    \subsubsection{Core Concepts}
        \begin{itemize}
            \item The array is recursively divided into two halves until each sub-array contains a single element.
            \item The sub-arrays are then merged back together in a sorted manner.
        \end{itemize}
    \subsubsection{Step-by-Step Explanation}
        \begin{itemize}
            \item Consider an array of n elements, a[0\ldots (n - 1)].
            \item \textbf{Step 1:} Recursively divide the array into halves until it reaches sub-arrays of size 1.
            \item \textbf{Step 2:} Merge the sorted sub-arrays back together by using two pointers to track the current position in the left and right sub-arrays, respectively.
            If there are remaining elements in the left sub-array, copy them to a temporary array. Repeat the same for the right sub-array.
            \item \textbf{Step 3:} Copy the sorted temporary array back to the original array a[left\ldots right].
            \item \textbf{Step 4:} Repeat the process until the entire array is merged and the recursion stack is empty.
        \end{itemize}
    \subsubsection{Pseudo Code}
        \begin{itemize}
            \item \textbf{Algorithm:} Merge Sort
            \item \textbf{Input:} An array A of n elements
        \end{itemize}
        \begin{verbatim}
            MergeSort(arr[], l, r):
                If l < r:
                    middle index: m = l + (r - l)/2
                    Call mergeSort(arr, l, m)
                    Call mergeSort(arr, m + 1, r)
                    Call merge(arr[], l, m, r) to merge the two halves

            merge(arr[], l, m, r):
                Set pos = left, leftPos = left, rightPos = mid + 1
                While leftPos <= mid and rightPos <= right:
                    tempArray[pos++] = min(a[leftPos], a[rightPos])
                    If a[leftPos] <= a[rightPos]: increment leftPos
                    else: increment rightPos
        \end{verbatim}
    \subsubsection{Complexity Analysis}
        \begin{itemize}
            \item \textbf{Time Complexity:}
            \begin{itemize}
                \item Best case: $O(n \log n)$
                \item Average case: $O(n \log n)$
                \item Worst case: $O(n \log n)$ 
            \end{itemize}
            \item \textbf{Space Complexity:} O(n) 
                \subitem Requires additional memory for the temporary array used in the merge process.
        \end{itemize}
        
    \subsubsection{Variants and Optimizations}
        \begin{itemize}
            \item \textbf{The Hybrid Sort (Mixing between Quick Sort and Merge Sort)}
            \begin{itemize}
                \item \textbf{Step 1:} Split the array [a] into [n] parts.
                \item \textbf{Step 2:} Select part [1]
                \item \textbf{Step 3:} Sort it using Quick Sort.
                \item \textbf{Step 4:} Select part [2] and implement the same process until part [n] is sorted.
                \item \textbf{Step 5:} Merge the sorted parts using Merge Sort.
            \end{itemize}
        \end{itemize}
